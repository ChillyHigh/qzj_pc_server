# Planning Setting

本文档记录规划层需要使用的比赛规则、场地坐标、目标点、机器人几何和约束。来源包括：

- `../scenes/qzj_field.xml`
- `../起重机规则.pdf`
- `planning_geometry.py`
- `../robot_kinematics.py`
- `../PROJECT_BOOK.md`

## 1. 坐标与单位

- 规划层统一使用场地世界坐标，单位为 m。
- `x` 轴从取货区指向放置区，左负右正。
- `y` 轴从下侧指向上侧，下负上正。
- 场地中心为 `(0, 0)`。
- 车身 `yaw` 使用 rad，逆时针为正。
- 五连杆 `q1/q2` 使用 rad，正方向与 `yaw` 一致，逆时针为正；`q=0` 时主动臂指向机器人局部 `+y`。
- MuJoCo 场地地面尺寸为 `4.0m x 2.0m`，规划可用区域按 `x in [-2.0, 2.0]`、`y in [-1.0, 1.0]` 处理。
- XML 中围栏中心位于场地边缘内侧，围栏宽度 `0.035m`，高度 `0.10m`。

## 2. 比赛任务

比赛开始前抽签产生两个输入：

- `pickup_assignment`：取货区 1、2、3 号位分别放置哪种货物。
- `drop_assignment`：放置区 4、5、6、7、8 号位分别摆放贴数字几的货箱。

货物编号约定：

| 编号 | 货物 | 目标货箱编号 |
|---:|---|---:|
| 1 | 黄豆 | 1 |
| 2 | 绿豆 | 2 |
| 3 | 白芸豆 | 3 |

放置区货箱编号 4 和 5 为空箱，不是目标货箱。规划层需要从 `drop_assignment` 反解：

```text
bean -> target_drop_position
```

示例：

```text
pickup_assignment = [3, 1, 2]
drop_assignment   = [4, 1, 2, 3, 5]

黄豆在取货位 2，目标在放置位 5
绿豆在取货位 3，目标在放置位 6
白芸豆在取货位 1，目标在放置位 7
```

## 3. 取货区 Setting

取货区有 3 个固定取货位，货箱和纸堆固定，不允许赛中调整。

| 取货位 | 世界坐标 `(x, y)` | 目标矩形半长 `(half_x, half_y)` | XML 高度信息 | 说明 |
|---:|---|---|---|---|
| 1 | `(-1.822, 0.500)` | `(0.105, 0.150)` | 纸堆高 `0.10m`，site z `0.1015m` | 上方取货位 |
| 2 | `(-1.822, -0.500)` | `(0.105, 0.150)` | 纸堆高 `0.05m`，site z `0.0515m` | 下方取货位 |
| 3 | `(-1.565, 0.000)` | `(0.105, 0.150)` | 纸堆高 `0.15m`，site z `0.1515m` | 中间靠右取货位 |

取货规划需要生成：

- 底盘停车姿态 `(x, y, yaw)`。
- 夹爪末端覆盖目标货箱内货物的姿态。
- 五连杆解 `h, q1, q2`。
- 夹爪自身姿态 `gripper_yaw` 和开合角 `gripper_opening`。
- 装载到 `upper_funnel / lower_funnel / gripper` 的动作。

规划层必须检查：

- 底盘外廓不压取货台、围栏、障碍物。
- 夹爪可达目标点。
- 夹爪进箱时，夹爪开口方向应与取货箱长边平行，避免斜插碰箱壁。
- 夹爪取豆时应支持 `h` 上升与 `gripper_opening` 关闭同步执行，避免先合爪后平移/上升时碰到存豆箱底部。
- `q1/q2` 在 `-pi..+pi`，`0` 时主动臂指向机器人局部 `+y`。
- `h` 在升降范围内。
- 取货动作不会把货物贴地拖拽。

## 4. 放置区 Setting

放置区有 5 个货箱位，4-8 号位置固定，货箱贴数字编号由抽签决定。

| 放置位 | 世界坐标 `(x, y)` | 目标矩形半长 `(half_x, half_y)` | 朝向 | 说明 |
|---:|---|---|---|---|
| 4 | `(1.605, 0.840)` | `(0.150, 0.105)` | 横向 | 上侧横向箱 |
| 5 | `(1.840, 0.400)` | `(0.105, 0.150)` | 竖向 | 右侧上箱 |
| 6 | `(1.840, 0.000)` | `(0.105, 0.150)` | 竖向 | 右侧中箱 |
| 7 | `(1.840, -0.400)` | `(0.105, 0.150)` | 竖向 | 右侧下箱 |
| 8 | `(1.605, -0.840)` | `(0.150, 0.105)` | 横向 | 下侧横向箱 |

放置动作可使用：

- 上漏斗放料。
- 下漏斗放料。
- 夹爪释放。

规划层需要为每个目标放置位生成候选：

```text
drop_position + carrier + base_pose + arm_pose/tool_pose
```

漏斗放料候选按漏斗出口贴近目标箱边缘生成。夹爪放料候选按夹爪覆盖目标箱区域生成。

## 5. 障碍、围栏与禁区

场地中有两个圆柱障碍物：

| 障碍物 | 世界坐标 `(x, y)` | 半径 | 高度 |
|---|---|---:|---:|
| obstacle_left | `(-1.000, 0.000)` | `0.051m` | `0.500m` |
| obstacle_right | `(1.000, 0.000)` | `0.051m` | `0.500m` |

规则要求：

- 货物需要按参考路径绕过障碍物。
- 行走方向不限制。
- 规则没有限定整机必须按同一路径绕障碍，但货物本身不能直接越过障碍路径约束。
- 货物通过障碍物时不能超过障碍物上表面，即货物高度应低于 `0.500m`。
- 比赛过程中货物必须离地，不能贴地拖拽。

规划层处理方式：

- 跨场动作统一建模为 `SMove(start_pose, end_pose)`。
- 跨场路径默认从取货区侧进入 S 型通道，绕开两个障碍。
- 底盘移动时默认工具处于安全携带状态，货物高度低于 `0.500m`。
- 如后续允许底盘移动时五连杆伸出，需要额外做轨迹包络检查；当前默认取放动作必须在底盘停稳后执行。

## 6. 起始与结束要求

起始区：

- XML 中起始区中心为 `(0.2, 0.0)`。
- 起始区尺寸为 `0.4m x 0.4m`。
- 起始区位于放置区一侧，左侧贴中心线。

规则要求：

- 准备阶段可移动部分在放置区一侧，不能越过或压住中心线。
- 所有吊具中心在地面投影需位于起始区，且吊具触地。
- 正式比赛时间为 4 分钟，中间不暂停。
- 完成全部搬运任务时，可移动部分必须全部位于放置区后，才能示意完成。
- 比赛结束时可移动部分未全部位于放置区，扣 10 分。

规划层处理方式：

- 初始 `RobotState` 应满足底盘在放置区侧，工具在起始区约束内。
- 完成动作后增加 `FinishPose`，确保底盘可移动部分全部在放置区侧。
- `FinishPose` 应进入计划评分，避免最后扣分。

## 7. 机器人规划几何

当前规划层使用简化几何。

底盘：

| 参数 | 值 |
|---|---:|
| 底盘左右长度 | `0.360m` |
| 底盘上下宽度 | `0.670m` |
| 规划半长 `CHASSIS_HALF_X` | `0.180m` |
| 规划半宽 `CHASSIS_HALF_Y` | `0.335m` |

五连杆：

| 参数 | 值 |
|---|---:|
| 电机 x 偏移 | `0.110m` in `planning_geometry.py` |
| 电机 y 半间距 | `0.080m` |
| 主动杆长度 | `0.160m` |
| 从动杆长度 | `0.240m` |
| `q1/q2` 范围 | `-pi..+pi`，`0` 时主动臂指向机器人局部 `+y` |
| `h` 范围 | `0..0.43153m` |

夹爪规划包络：

| 参数 | 值 |
|---|---:|
| 完全张开尺寸 | `0.130m x 0.255m` |
| `GRIPPER_HALF_X` | `0.065m` |
| `GRIPPER_HALF_Y` | `0.1275m` |

其中 `0.255m` 沿机器人局部 `y` 方向，`0.130m` 沿机器人局部 `x` 方向。`GRIPPER_HALF_X/HALF_Y` 表示夹爪全开矩形包络的半长。

夹爪新增两个控制量：

| 控制量 | 单位 | 含义 |
|---|---|---|
| `gripper_yaw` | rad | 夹爪绕末端竖直轴的朝向，用于让夹爪开口方向平行于取货箱或放置箱 |
| `gripper_opening` | rad | 夹爪开合角，`0` 表示完全闭合，最大值表示完全张开 |

规划层需要在夹爪候选中同时确定 `gripper_yaw` 和 `gripper_opening`。取货阶段允许生成“合爪 + 上升”的同步动作段，即同一动作段内 `h` 增大、`gripper_opening` 减小。

漏斗规划出口：

| 参数 | 值 |
|---|---:|
| `UPPER_FUNNEL_OUTLET` | `(-0.070, +0.335)` |
| `LOWER_FUNNEL_OUTLET` | `(-0.070, -0.335)` |

注意：

- `planning_geometry.py` 与 `robot_kinematics.py` 均应保持该角度约定。

## 8. 规划输入与输出

规划输入：

```python
PlanningInput(
    pickup_assignment: list[int],  # length=3, values are bean ids 1..3
    drop_assignment: list[int],    # length=5, values are box ids 1..5
    current_state: RobotState,
)
```

规划输出：

```python
Plan(
    actions: list[ActionBlock],
    estimated_time: float,
    pickup_order: list[int],
    drop_order: list[int],
    container_assignment: dict[int, Carrier],
)
```

Carrier：

```text
upper_funnel
lower_funnel
gripper
```

## 9. 规划约束

硬约束：

- 三种货物必须分别进入对应目标编号货箱。
- 不能搬运货箱。
- 不能投掷。
- 不能混装。
- 货物必须离地搬运。
- 底盘与接地货物部分不能超出比赛场地。
- 底盘外廓不能撞围栏、取货台、放置箱、障碍柱。
- 取放动作的 arm 姿态必须可达。
- `q1/q2/h/yaw` 必须满足机构限制；`q1/q2` 的机械角范围为 `-pi..+pi`。
- 正式比赛时长为 `240s`。

软约束：

- 减少跨场次数。
- 减少底盘平移距离。
- 减少大角度 yaw 切换。
- 右侧放货顺序倾向高 y 到低 y。
- 取货区优先利用相邻纸堆形成的停靠区，减少底盘切换。
- 取货后优先一趟跨场完成三种货物。

## 10. 评分模型

规划层优化目标：

```text
score = estimated_finish_time + risk_penalty + yaw_switch_penalty + recovery_penalty
```

第一版可使用：

```text
score = estimated_finish_time
```

计分相关策略判断：

- 搬运分达到 100 分后才有速度分，因此优先保证三种目标货物有效搬运。
- 若机构容量支持，优先一趟携带三种货物。
- 速度分为 `240 - 实际完成时间`，规划估时应覆盖全部取货、跨场、放货和结束停车动作。
- 完成时可移动部分必须在放置区，否则扣 10 分；结束停车是计划的一部分。
